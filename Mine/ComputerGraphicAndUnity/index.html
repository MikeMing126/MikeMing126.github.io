前言

主要还是结合之前学校计算机图形学的课程、Game101课程和Unity相关知识，进行重新梳理。

计算机图形学

基础

点乘 叉乘

点乘 - 两向量间夹角，以及向量投影

  • 可以用于评估向量的方向、相似度（Close）等

叉乘 - 两向量的垂直向量，值为两向量构成四边形的面积

  • 注意左手坐标系用左手法则，右手坐标系用右手法则。
  • 用于判定向量左右/内外关系
  • 行列式计算叉乘

Unity 渲染管线

Unity Shader（零）-顶点着色器和片元着色器
猴子也能看懂的渲染管线（Render Pipeline）
GPU 渲染管线与着色器
图形渲染管线详细梳理

基础流程

CPU应用程序端渲染逻辑 - GPU渲染管线 - FrameBuffer

应用阶段 Application Stage

CPU主导

  • 元素剔除
    • GPU裁剪阶段也有剔除。但是在CPU阶段剔除可以减少传输的数据。
    • 依据FOV、FarClip、NearClip构成视锥体，并进行视锥体剔除
  • 排序
    • 基于RenderQueue渲染队列来进行排序处理，确保正确的渲染顺序并提高渲染效率。（深度测试/写入的开关、多个Pass时减少切换优化性能）
    • 半透明一般RenderQueue>不透明，不透明相同的RenderQueue则按照摄像机距离从前到后进行排序。半透明相同的RenderQueue则按照摄像机距离从后到前进行排序。
    • Camera.Depth（高的后渲染） > 材质类型(是否透明,具体由RQ判断) > RenderQueue > sorting layer > order in layer > 相机距离z。 sorting layer与order in layer适用于基于摄像机/世界空间的Canvas。以及Sprite
    • 参考文章Unity中SortingLayer、Order in Layer和RenderQueue的讲解 和 U3D sorting layer, sort order, order in layer, layer深入辨析
  • 数据的准备
    • 模型相关的: 顶点、法线、UV、切线、顶点色、索引列表
    • 灯光：灯光数据、材质信息
    • 变换矩阵：世界变换矩阵、依据摄像机构建的VP矩阵
  • 渲染状态的设置和绘制调用
    • SetPass Call
    • Draw Call

Vertex Shader 顶点着色器

  • 矩阵变换的计算
  • 计算光照公式生成逐顶点颜色（可选）
  • 生成/变换纹理坐标

在VertexShader这里其实还有曲面细分着色器Tessellation Stage和几何着色器Geometry Shader。
不过倒不是所有的都支持 暂且略过。

计算机图形学二：视图变换(坐标系转化，正交投影，透视投影，视口变换)

Rasterization 光栅化

将三角形划分对应到单元格区域。
将图元变为片元。
3D游戏对应的片元大部分情况下会大于屏幕的宽和高。每个三角形都会计算片元并对应到屏幕坐标。

Fragment Shader 片元着色器

或称为Pixel Shader

  • 光照和颜色计算
  • 获取纹理，进行纹理映射
  • 往像素填充颜色值，处理混合以及特殊效果

Per-Fragment Operations 逐片元操作

在Fragment Shader到Frame Buffer之间。逐片元操作是可以配置但不可编程的，也就是说是由管线/硬件自身规定好的，我们只能对里边的内容进行配置。

进行测试与合并的工作。

  • 裁剪测试 Scissor Test
  • 透明度测试 Alpha Test （后面OpenGL改了不支持，相关逻辑可以放到Fragment Shader中去实现）
    • 提前设置一个透明度预值，只能实现不透明效果和全透明效果。
  • 模板测试 Stencil Test
  • 深度测试 Depth Test

然后进行Blending（透明度混合），并在进行一系列其他输出后，输出到帧缓冲区。

模板测试和深度测试

3.1 模板测试和深度测试
【Unity】深度图（Depth Texture）的简单介绍

处理阶段都是位于逐片元阶段。

模板测试

referenceValue：当前模板缓冲片元的参考值
stencilBufferValue：模板缓冲区的值
comparisionFunc：比较函数

模板缓冲区：FrameBuffer 可以为屏幕上每一个像素点保留一个无符号整数值。

  • 最重要(用来比较的）两个值：当前模板缓冲区值（StencilBufferValue）、模板参考值（ReferenceValue）
  • 模板测试主要就是对这两个值进行特定的比较操作，例如Never、Always、Equal等，具体参考上文的表格
  • 模板测试后，要对模板缓冲区的值进行更新操作，例如Keep，Replace等，具体参考上文表格
  • 模板测试之后，可以通过不同的结果对模板缓冲区做不同的更新操作，例如模板测试成功的操作Pass、模板测试失败的操作Fail、深度测试失败的操作ZFail、还有正对正面和背面更新操作Passback，Passfront，Failback等…

应用

描边:
【Unity URP】2种描边方案：模板测试和正面剔除
【Unity3D】基于模板测试和顶点膨胀的描边方法

深度测试

处理物体的遮挡关系。

总体上还是和模板测试比较类似的。

作用

  • 控制渲染顺序
    • 画家算法：从远处开始画，近处的东西叠加在上面（存在多画无用的问题）
    • ZBuffer算法：通过深度缓冲区来控制渲染顺序
  • 控制Z-Buffer对深度的存储
    • 什么时候更新、什么时候使用深度缓冲区。Z Test和Z Write。
  • 控制不同类型物体的渲染顺序
    • 透明物体
    • 不透明物体
    • 渲染队列
  • 减少Overdraw
    • Early-Z
    • Z-cull
    • Z-check

ZBuffer

深度缓冲区 Z-Buffer
颜色缓冲区：最终显示的区域。

Z Write：
开启情况下，深度测试通过会写入深度缓冲区。（不开启时，通过时，不写入深度缓冲区，但仍然会写入颜色缓冲区）。
默认情况下为开启，且比较为 ZTest LEqual（一开始初始值为无限大）。

渲染队列

按照渲染顺序从先到后排序，队列数越小，越先渲染；反之同理。
Unity中设置渲染队列：

  • 语法：Tags { “Queue” = “Transparent”}
  • 默认是Geometry

Unity中不透明物体的渲染顺序：从前往后，也就是说深度小的先渲染，其次再渲染深度大的
Unity中透明物体的渲染顺序：从后往前（类似画家算法，会造成OverDraw）

Early-Z

传统的渲染管线中，ZTest是在Blending阶段，这时进行深度测试的话，所以对象的像素着色器都会计算一遍，没有性能提升，只是为了得到正确的效果，造成了大量的无用计算。（深度测试失败的片元是已经经过计算的，到这一步不通过被抛弃，那么前边的计算就是无用功了）

为了减少这些不必要的计算，现代GPU运用了Early-Z技术，在顶点和片元阶段之间（光栅化之后，片元着色器之前）进行一次深度测试。

如果这次深度测试失败，那就不用在片元着色器中作无关紧要的计算了，这样一来就会带来性能提升。

最终的ZTest仍然要进行，以保证正确的遮挡关系。

深度值

坐标在经过MVP矩阵变换后，会获得一个深度值。
其深度值为z/w

应用

  • 透视物体（使用ZTest - Greater）
  • shadow map阴影贴图实现原理

Mipmap

Mipmap详解
UnityDoc Mipmap

渲染的时候需要进行纹理映射。
而在3D场景中，随着物体的远近，其映射会有所不同。

邻近过滤：

  • 选择离中心点最近的纹理坐标的像素作为目标像素（直接取一个像素值作为目标值）

线性过滤

  • 基于纹理坐标附近的纹理像素进行插值计算，近似得到纹理像素之间的颜色。

但目标很远的时候，两种效果都不好。会有对应的锯齿(摩尔纹)问题

Mipmap：
生成一系列图像。创建多级渐远的纹理。

优点：

  • 可以解决锯齿问题。
  • 性能更好，可以在更小的Texture上进行采样。(对于远距离对象，采样频率高和数据频率高而造成Textecute的cache命中率不高)

缺点：

  • 占用更多内存（多33%）

Unity中的Mipmap

原理和上面所述一致。适用于在3D场景中渲染的对象。对于UI这种无缩放的，则没必要使用。

Unity自动生成Mipmap需要其分辨率为2的幂。
相关的Mipmap设置可以看到Texture的Import Settings的Advanced。

GPU采用的Mipmap级别：
依赖于当前像素对应的UV坐标以及GPU内部的两个值：DDX和DDY。
DDX和DDY提供当前像素旁边和周围像素的UV信息（包含距离和角度）

  • 与相邻像素之间距离越大、角度越极端 - 选择分辨率较低的mipmap级别。

还可以通过两个级别的Mipmap进行混合（Trilinear Filtering 在两层Mipmap上进行双线性过滤，然后再进行两个结果的线性插值）。让Mipmap之间的过渡更为自然。从一个Mipmap采取一定百分比的纹理信息，然后剩余的从另外一个取。

Mipmap bias

可以调整GPU选择的Mipmap级别。
比如默认选择Mipmap3级，使用Bias = -2，则会选择分辨率更高的Mipmap1级。

此外，也可以用于调整两个级别Mipmap混合的百分比。如原来为0.5,0.5，添加0.2的bias则会变为0.7和0.3.

有Global Mipmap Bias与Texture自己的Mipmap Bias(在全局的基础上加上/减去该偏差)。
或者在Shader中为单个纹理处理Bias。

Mipmap Streaming

可以读读这篇文章
浅谈Unity纹理串流系统Mipmap Streaming System

在默认情况下，不使用MipmapStreaming，则会将所有Level的Mipmap加载到显存中。

对于纹理，计算Mipmap对应的层级。并只加载对应特定层级的Mipmap（从CPU传到GPU）。按需加载。

一般情况下都是尽可能加载高分辨率的Mipmap，但也可以设置内存限制，使用较低Level的Mipmap。

Mipmap在加载到GPU上后会进行缓存，避免重复从CPU和硬盘加载对应Level的Mipmap。

Limitations:

  • 不支持Terrain的Texture（始终需要最高分辨率级别）
  • 不支持纹理数组、cubemap数组或3D Texture
  • 注意API调用，对于Graphics.DrawMeshNow等API没有对应的Mipmap信息。使用Texture2D.requestedMipmapLevel设置对应的级别。
  • Unity might not be able to calculate the correct mipmap level for textures that don’t use the special _ST property for texture tiling and offset.

计算机图形学算法

判断两个矩形是否相交

坐标轴平行的情况

边界条件判断法，一个矩形完全位于另一个矩形的左侧、右侧、上方或下方。
准确说是通过两个矩形的左上角、右下角来进行判断。

不相交条件（任一成立即不相交）：

  • 矩形1在矩形2左边：x1_max <= x2_min
  • 矩形1在矩形2右边：x2_max <= x1_min
  • 矩形1在矩形2上边：y1_max <= y2_min
  • 矩形1在矩形2下边：y2_max <= y1_min

坐标轴不平行的情况

也可以看作是两个凸多边形是否相交的问题。

轴分离定理 SAT

碰撞检测算法之分离轴定理
适用于凸多边形
对于矩形而言：
如果存在一个轴，使得两个矩形在该轴上的投影不重叠，则它们不相交；如果对于所有可能的轴，两个矩形的投影都重叠，则它们相交。

  • 取其每条边的法向量
  • 将两个矩形的所有顶点投影到当前轴上
  • 找出投影的最小值和最大值，形成投影区间
  • 如果在任何轴上，两个投影区间不重叠，则矩形不相交
