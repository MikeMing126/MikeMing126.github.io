Batching Meshes - 静态和动态批处理

Draw call batching
Introduction to batching meshes

基础条件和说明

  • 相同材质
  • 可以使用MaterialPropertyBlock，这个不会打断合批。(但是对SRP会移除SRP Batcher的兼容性)
  • 可以减少DrawCall的数量

支持:

  • Mesh Renderers
  • Trail Renderers
  • Line Renderers
  • Particle Systems
  • Sprite Renderers

不支持

  • Skinned Mesh Renderers

Unity仅将同类型的Renderers进行批处理。

对于透明材质(Transparent shaders)，批处理的效果弱于非透明的。

When you access shared material properties from a C# script, make sure to use Renderer.sharedMaterial and not Renderer.material. Renderer.material creates a copy of the material and assigns the copy back to the Renderer. This stops Unity from batching the draw calls for that Renderer.

注意使用Renderer.sharedMaterial而非Renderer.material，后者会创建材质副本。

静态批处理 Static Batch

存在额外的内存和存储开销。
在构建和运行时进行静态批处理。

  • 如果构建前在场景中 - 使用编辑器来进行批处理
    ProjectSetting - Player - Static Batching
  • 运行时创建的游戏对象及网格 - 需要运行时API。

其他条件:

  • GO Active
  • 有Mesh Filter，其Active且引用了Mesh
  • 引用的Mesh顶点数大于0且尚未和另外一个网格合并
  • 有Mesh Renderer，其Active，且使用的材质没有 DisableBatching
  • 合批的对象使用相同的顶点属性。
  • 对象在static标签中设置为Batching Static。

相关观测见Stats中的Batches和Saved By batching。

运行时处理

StaticBatchingUtility。使用运行时API时，无需设置PlayerSettings中的Static Batching。

性能问题

使用静态批处理需要额外的 CPU 内存来存储组合的几何体。如果多个游戏对象使用相同的网格，Unity会为每个游戏对象创建网格的副本，并将每个副本插入组合的网格中。这意味着相同的几何体会在组合的网格中多次出现。

很多对象的时候考虑GPUInstance。

可以在Memory Profile中的Mesh看到静态批处理生成的Combined Mesh。

动态批处理

Dynamic batching for meshes was designed to optimize performance on old low-end devices.

网格的动态批处理通过在 CPU 上（而不是 GPU 上）将所有顶点变换到世界空间来实现。这意味着，只有当变换工作比执行绘制调用占用的资源更少时，动态批处理才是一种优化。
官方在Unity6相关文档说到，大多数情况下不再建议使用Dynamic Batching。

动态批处理不支持transform中scale为负值的对象和scale为正值的对象进行合批。

For:

  • Built-in Particle Systems
  • Line Renderers
  • Trail Renderers

动态批处理由Unity自动处理。

GPU Instancing

For Same Material and Same Mesh。Single Draw Call。

GPU Instancing与SRP Bathcer不兼容，后者优先级更高。

要求和说明：

  • 材质支持GPU Instancing
  • 可以使用 MaterialPropertyBlock来控制对应的区分属性
  • Mesh存在限制，必须满足以下之一：
    • 对于MeshRenderer Component/Graphics.RenderMesh,Unity将其加入列表中，并检查哪些网格可以实例化。不支持SkinnedMeshRenderers,或者是在SRP Batcher Compatible的GO上的MeshRenderer.
    • 对于Graphics.RenderMeshInstanced或Graphics.RenderMeshIndirect，这些都是单独绘制调用，不会进行合并。

启用GPUInstancing:
在材质设置GPU Instancing即可。

不过对于非常少顶点数的Mesh，用GPU Instancing反而可能会降低效率。建议是256个顶点以上才考虑用
Troubleshooting GPU instancing

SRP Batching

可以基于这篇文章先做简要的了解
Unity渲染优化的4种批处理：静态批处理，动态批处理，SRP Batcher 与 GPU Instancing

本质上并不会减少DrawCall,但是可以减少SetPass.
相较于DrawCall,SetPass的开销会更高。

要求和说明：

  • 支持Mesh、Skinned Mesh Renderer
  • 不支持Particle
  • 不支持Material Property Block。

SRP中，材质球在显存中占用固定的CBuffer(Constant Buffer)。其分为了Per Object Large Buffer用于存储对象属性和Material CBuffer用于存储材质信息。仅当材质信息改变时，才需要对Material CBuffer进行更新处理。

  • 中断SetPass的条件由Material Break 变为 Shader Variant Break。
  • 减少了CBuffer的相关操作。
