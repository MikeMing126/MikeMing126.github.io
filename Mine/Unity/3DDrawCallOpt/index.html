Unityд»…е°ҶеҗҢзұ»еһӢзҡ„RenderersиҝӣиЎҢжү№еӨ„зҗҶгҖӮ

еҜ№дәҺйҖҸжҳҺжқҗиҙЁ(Transparent shaders)пјҢжү№еӨ„зҗҶзҡ„ж•ҲжһңејұдәҺйқһйҖҸжҳҺзҡ„гҖӮ

When you access shared material properties from a C# script, make sure to use Renderer.sharedMaterial and not Renderer.material. Renderer.material creates a copy of the material and assigns the copy back to the Renderer. This stops Unity from batching the draw calls for that Renderer.

жіЁж„ҸдҪҝз”ЁRenderer.sharedMaterialиҖҢйқһRenderer.materialпјҢеҗҺиҖ…дјҡеҲӣе»әжқҗиҙЁеүҜжң¬гҖӮ

дҪҝз”ЁйқҷжҖҒжү№еӨ„зҗҶйңҖиҰҒйўқеӨ–зҡ„ CPU еҶ…еӯҳжқҘеӯҳеӮЁз»„еҗҲзҡ„еҮ дҪ•дҪ“гҖӮеҰӮжһңеӨҡдёӘжёёжҲҸеҜ№иұЎдҪҝз”ЁзӣёеҗҢзҡ„зҪ‘ж јпјҢUnity дјҡдёәжҜҸдёӘжёёжҲҸеҜ№иұЎеҲӣе»әзҪ‘ж јзҡ„еүҜжң¬пјҢе№¶е°ҶжҜҸдёӘеүҜжң¬жҸ’е…Ҙз»„еҗҲзҡ„зҪ‘ж јдёӯгҖӮиҝҷж„Ҹе‘ізқҖзӣёеҗҢзҡ„еҮ дҪ•дҪ“дјҡеңЁз»„еҗҲзҡ„зҪ‘ж јдёӯеӨҡж¬ЎеҮәзҺ°гҖӮ

еҫҲеӨҡеҜ№иұЎзҡ„ж—¶еҖҷиҖғиҷ‘GPUInstanceгҖӮ

еҠЁжҖҒжү№еӨ„зҗҶ

Dynamic batching for meshes was designed to optimize performance on old low-end devices.

зҪ‘ж јзҡ„еҠЁжҖҒжү№еӨ„зҗҶйҖҡиҝҮеңЁ CPU дёҠпјҲиҖҢдёҚжҳҜ GPU дёҠпјүе°ҶжүҖжңүйЎ¶зӮ№еҸҳжҚўеҲ°дё–з•Ңз©әй—ҙжқҘе®һзҺ°гҖӮиҝҷж„Ҹе‘ізқҖпјҢеҸӘжңүеҪ“еҸҳжҚўе·ҘдҪңжҜ”жү§иЎҢз»ҳеҲ¶и°ғз”ЁеҚ з”Ёзҡ„иө„жәҗжӣҙе°‘ж—¶пјҢеҠЁжҖҒжү№еӨ„зҗҶжүҚжҳҜдёҖз§ҚдјҳеҢ–гҖӮ
