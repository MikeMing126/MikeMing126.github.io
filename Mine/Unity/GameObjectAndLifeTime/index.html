GameObject And LifeTime

|总字数:979|阅读时长:3分钟|浏览量:

前言 - Fundamental Unity types

在Unity6的官方文档中，提到了以下类型，这里做简要概述：

  • Object
    UnityEngine.Object. 一般不会直接继承自UnityEngine.Object。而是选择更符合目标的类。如MonoBehaviour与ScriptableObject。
  • MonoBehaviour
    继承自Object。可以添加到Object的自定义组件。
  • ScriptableObject
    继承自Object。可以存储序列化数据的自定义资产。不会作为组件附加到GameObjects,而是作为assets存在于项目中,独立于GameObject。主要价值在于数据存储。
  • Unity Attributes
    元数据标记，用于指示特殊行为。参考API UnityEngine-Attributes与UnityEditor-Attributes

游戏对象 GameObjects

Represent everything in your game

始终附加一个Transform组件。
后简称GO。

操作

Activate

  • SetActive方法与activeSelf属性。
  • 但是由于其Activate受到父对象影响，要检测对象是否处于活跃状态还是要看activeInHierarchy属性。

Tag

可以给GO添加标签。一个游戏对象只能有一个标签。标签本身也没有啥层级关系。
可以认为是一个引用词(“Reference Word”)

使用标签查找对象

  • FindGameObjectWithTag
  • FindGameObjectsWithTag

不能查找被隐藏的物体。
必须在使用标签之前在标签管理器中声明标签。如果标签不存在，或者如果提供了空字符串或 null 作为 tag 参数，则将抛出 UnityException。

其排序还是挺奇怪的，是静态函数，和Script所在GameObject无关。

静态游戏对象 Static GameObject

在运行时不移动的游戏对象。编辑器中设置Static。
可以进行预计算，并节省部分系统的开销。

场景切换

在场景切换时，GO会销毁
使用DontDestroyOnLoad：

DontDestroyOnLoad(this.gameObject);

脚本生命周期

Unity以循环方式运行，Game(Player) Loop。

Traditional component-based Unity projects use MonoBehaviour script components to hook into the Player loop through a series of built-in callbacks called event functions

前言

Unity6 - Programming in Unity 提到了Object-oriented development。

传统面向对象开发：
Code is organized around GameObjects,whose behavior is controlled by attached MonoBehaviour components.

然而，传统的面向对象开发在进行规模化时，可能存在性能问题。
替代方案：面向数据开发 Data-oriented development. 代码和数据分离，最大限度提高CPU缓存效率和并行性。ECS框架。

ECS并不使用MonoBehaviour，而是使用组件来处理数据，系统来处理逻辑。DOTS(Data-oriented Technology Stack).
DOTS官方文档

两种方式可以结合使用。

MonoBehaviour

官方Doc

事件函数周期：

  • Awake 出生时调用 类似于构造函数。

    Unity calls Awake when an enabled script instance is being loaded.

  • OnEnable 每次激活时调用

    This function is called when the object becomes enabled and active.

  • Start 第一次Update前调用，仅一次(如果在Awake中直接SetActive禁用，则不会调用。)

    Start is called on the frame when a script is enabled just before any of the Update methods are called the first time.

  • FixedUpdate 物理帧更新 固定间隔(Timestep)执行。

    Frame-rate independent MonoBehaviour.FixedUpdate message for physics calculations.

  • Update 逻辑帧更新
  • LateUpdate 动画更新后调用
  • OnDisable 禁用时调用（如果直接调用Destroy，已激活的GO也会先调用Disable）

    This function is called when the behaviour becomes disabled.

  • OnDestroy 销毁时调用

    Destroying the attached Behaviour will result in the game or Scene receiving OnDestroy.

其中，OnEnable和OnDisable和循环的关联性没那么大。
比如在Start过程中，可以直接销毁物体，也会调用到OnDisable然后OnDestroy。

Unity设计的MonoBehaviour，其不能new而是挂载。构造相关的逻辑处理可以放于Awake。

生命周期函数支持继承和多态
