C# 简介

由微软开发。专为公共语言基础结构（CLI）设计的。CLI 由可执行代码和运行时环境组成，允许在不同的计算机平台和体系结构上使用各种高级语言。

面向对象编程。

C# 数据类型

引用类型 Reference types

引用类型继承自System.Object.
对变量的引用。指向内存位置。
内置引用类型：

  • object 对象类型
    C# 通用类型系统（Common Type System - CTS）中所有数据类型的终极基类。Object 是 System.Object 类的别名。所以对象（Object）类型可以被分配任何其他类型（值类型、引用类型、预定义类型或用户自定义类型）的值。但是，在分配值之前，需要先进行类型转换。当一个值类型转换为对象类型时，则被称为 装箱；另一方面，当一个对象类型转换为值类型时，则被称为 拆箱。
  • dynamic 动态类型
    在动态类型存储任何类型的值。变量的类型检查在运行时发生。(可以改变类型)
  • string 字符串类型
    可以通过在字符串前加上 @逐字字符串 表示字符串中的特殊字符（如反斜杠 \）不需要转义和多行处理。 用户定义的引用类型：
  • class
  • interface
  • delegate

指针类型
同C++，但只能在unsafe上下文中使用。

装箱和拆箱

C#装箱和拆箱（Boxing 和 UnBoxing）
简而言之： 值类型和引用类型的转换
装箱：值->引用

  • 在堆中分配内存，创建一个新的对象实例。
  • 将值类型的数据复制到堆中的对象中。
  • 返回该对象的引用（即 object 类型）。
    拆箱：引用->值
  • 需要显式操作。手动进行类型转换。
  • 拆箱失败：如果类型不匹配，会抛出 InvalidCastException
    .NET中，数据类型划分为值类型和引用(不等同于C++的指针)类型，与此对应，内存分配被分成了两种方式，一为栈，二为堆，注意：是托管堆。

值类型只会在栈中分配。 引用类型分配内存于托管堆。（托管堆对应于垃圾回收。）

常量

基础数据类型。

整数常量

整数常量可以是十进制、八进制或十六进制的常量。前缀指定基数：0x 或 0X 表示十六进制，0 表示八进制，没有前缀则表示十进制。
整数常量也可以有后缀，可以是 U 和 L 的组合，其中，U 和 L 分别表示 unsigned 和 long。后缀可以是大写或者小写，多个后缀以任意顺序进行组合。

浮点数常量

使用浮点形式表示时，必须包含小数点、指数或同时包含两者。使用指数形式表示时，必须包含整数部分、小数部分或同时包含两者。有符号的指数是用 e 或 E 表示的。

常量定义 const

使用const来定义常量。核心特性是：值在编译时确定，且在程序运行期间不可修改。

仅支持数字、布尔值、字符或者是null引用。

**类成员变量如果声明为const，则其默认是静态的。**需要通过类名去访问。

比较类似的还有readonly。

拓展:readonly
对比维度 const readonly
初始化时机 编译时必须初始化（声明时赋值）。 运行时初始化（可声明时或构造函数中赋值）。
值类型 仅支持基本类型、字符串或 null。 支持所有类型（包括复杂类型和自定义类）。
静态性 默认是静态的（static），通过 类名.字段名 访问。 可以是实例字段（需通过对象访问）或静态字段。
灵活性 值固定，无法动态计算。 支持运行时动态计算（如依赖 DateTime.Now）。
重新编译需求 如果修改 const 的值，所有引用该常量的程序集必须重新编译。 修改 readonly 的值无需重新编译引用代码（只需更新程序集）。

封装

把一个或多个项目封闭在一个物理的或者逻辑的包中”。在面向对象程序设计方法论中，封装是为了防止对实现细节的访问。

C# 访问修饰符

  • public
  • protected internal: 访问修饰符允许在本类,派生类或者包含该类的程序集中访问。即protected和internal的并集。
  • internal：同一个程序集的对象可以访问
  • protected
  • private:仅限类型内部。默认访问级别。

和C++不同，每个成员需要显式指定访问修饰符，或者依赖默认值。

程序集

定义

程序集是 .NET 应用程序的构建、部署和版本控制的基本单元。它可以简单理解为一个 .exe（可执行文件）或一个 .dll（类库文件）。

  • .exe：可执行程序集，包含入口点（Main 方法）。
  • .dll：类库程序集，包含可重用的代码和资源，供其他程序集调用。
程序集包含什么？

一个程序集远不只是一堆编译后的代码。它是一个自描述的逻辑单元，通常包含以下内容：

  • 中间语言 (IL) 代码：由你的C#代码编译而成。
  • 元数据 (Metadata)：描述程序集本身及其包含的所有类型信息（如类名、方法名、参数等）。这就是“自描述”的核心。
  • 清单 (Manifest)：程序集的“说明书”，包含程序集本身的元数据，如：
    • 程序集名称、版本号、文化信息、强名称。
    • 它所依赖的其他程序集的列表。
    • 它包含的所有文件和资源的列表。
  • 资源文件（可选）：如图标、图像、本地化字符串等。
程序集 vs. 命名空间 (Namespace)

这是一个常见的混淆点。程序集是物理上的部署单位，而命名空间是逻辑上的组织单位。

  • 一个程序集中可以包含多个命名空间。
    • 例如，System.dll 程序集中包含了 System、System.IO、System.Collections 等多个命名空间。
  • 一个命名空间中的类也可以分布在多个程序集中。
    • 例如，System.Linq 命名空间中的类主要存在于 System.Core.dll 中，但其他程序集也可能包含该命名空间下的类。

简单比喻：

  • 程序集就像一个“书”（.exe或.dll文件）。
  • 命名空间就像书中的“章节”。
  • 类就像章节中的“具体内容”。

internal 的可见性边界是由“书”（程序集）来划分的，而不是由“章节”（命名空间）来划分的。

结构体 Struct

结构体（struct）是一种值类型（value type），用于组织和存储相关数据。
不能直接设置为 null：因为 null 是引用类型的默认值，而不是值类型的默认值。若要支持null，使用? Nullable。

  • 结构可带有方法、字段、索引、属性、运算符方法和事件，适用于表示轻量级数据的情况，如坐标、范围、日期、时间等。
  • 结构可定义构造函数，但不能定义析构函数。但是，您不能为结构定义无参构造函数。无参构造函数(默认)是自动定义的，且不能被改变。
  • 与类不同，结构不能继承其他的结构或类。但可实现一个或多个接口。
  • 结构不能作为其他结构或类的基础结构。
  • 结构成员不能指定为 abstract、virtual 或 protected。
  • 当您使用 New 操作符创建一个结构对象时，会调用适当的构造函数来创建结构。与类不同，结构可以不使用 New 操作符即可被实例化。
  • 如果不使用 New 操作符，只有在所有的字段都被初始化之后，字段才被赋值，对象才被使用。
  • 结构变量通常分配在栈上，这使得它们的创建和销毁速度更快。但是，如果将结构用作类的字段，且这个类是引用类型，那么结构将存储在堆上。
  • 结构默认情况下是可变的，这意味着你可以修改它们的字段。但是，如果结构定义为只读，那么它的字段将是不可变的。
  • 结构体中声明的字段无法赋予初值。

枚举 Enum

枚举是一组命名整型常量。是值类型。枚举包含自己的值，且不能继承或传递继承。

接口Interface

接口定义了属性、方法和事件，这些都是接口的成员。

  • 接口只包含了成员的声明。从 C# 8.0 开始，接口可以包含默认实现（Default Interface Methods）.
  • 接口不能包含字段。（如 int）
  • 接口的成员默认是public和abstract的，且不能使用修饰符。实现接口的类对应的方法也必须是public。

通常命名多一个I。

关于C#8.0 支持的接口默认实现

虽然支持接口默认实现，但是实现接口的类不能直接调用接口的方法。需要将对象转为对应的接口对象才能调用。

类Class

<access specifier> class class_name

访问标识符 <access specifier> 指定了对类及其成员的访问规则。如果没有指定，则使用默认的访问标识符。类的默认访问标识符是 internal，成员的默认访问标识符是 private。

C# 中 类（class） 的访问标识符只能是以下两种：
public：类可以在任何位置访问（包括其他程序集）。
internal：类只能在当前程序集（项目）内访问。

关键字

abstract

用于声明抽象类或抽象方法。
抽象类不能被实例化，必须被继承。
抽象方法没有具体实现（无方法体），子类必须重写（override）所有抽象方法。

抽象方法只能在抽象类中声明，其不能是private，不能有默认实现。当然，抽象类可以没有抽象方法。

抽象属性。Property，见属性部分。

virtual

用于声明可重写的方法、属性或事件。
子类可以通过override重写virtual方法。（和C++不一样，override要放前面）

public override void TestFunc1()

  • virtual 方法必须有默认实现。
  • 不能与 static、private、sealed 同时使用。
  • 子类override后不需要也不能再加virtual

C#中的析构函数在多态中无需显式也不能声明为virtual。C#的析构函数默认是虚函数。
将析构函数声明为virtual算语法错误。

override

它允许子类 重写（覆盖） 基类中已声明为 virtual、abstract 或 override 的方法、属性、索引器或事件，从而提供新的实现。

  • 重写方法必须与基类方法具有相同的 方法名、返回类型、参数列表。
  • 子类方法的访问级别不能比基类更严格（例如，基类为 protected，子类不能为 private）。
  • override 方法不能使用 new、static 或 virtual 修饰符。

区分方法隐藏 new。
使用override为多态，基类引用调用子类方法。
而new时静态绑定。编译时确定。基类引用，调用的还是基类方法。

sealed

用于限制类的继承或方法的重写，主要目的是增强代码的安全性和性能。

用于类时

阻止继承。但不能与abstract同时使用。

sealed class A{}; // class B:A{} not allow

用于方法时

阻止重写。必须要与override一起用。

using

  1. 引入命名空间。可以直接使用该命名空间下的类型，而无需完全限定名称。
  2. 为命名空间或类型定义别名。 但是需要注意，放在文件开头。

    using Project = PC.MyCompany.Project;

  3. using static 指令：指定无需指定类型名称即可访问其静态成员的类型

    using static System.Math;var = PI; // 直接使用System.Math.PI\

  4. 用作资源管理
    • 自动释放资源：确保实现了IDisposable接口的对象在其生命周期结束时被正确释放（调用 Dispose() 方法）。
    • 异常安全：即使在 using 块内发生异常，资源仍会被释放。
    • 简化代码：无需手动调用 Dispose() 或 Close()，代码更简洁。
    • IDisposable 是 .NET 中用于资源清理的标准接口，包含 Dispose() 方法。通常管理非托管资源

yield

C# 中的yield return机制和原理
简化迭代器的实现。本质上yield还是迭代器。
用于在迭代器块中向枚举器对象提供值或发出迭代结束的信号。它与 return 和 break 语句结合使用。

yield实现延迟执行和按需生成。使用yield时，本质上是由C#生成状态机。
使用yield关键字的函数必须返回IEnumerable 或 IEnumerator 类型（包括它们的泛型版本 IEnumerable 和 IEnumerator）

  • yield return ;：在迭代过程中，返回序列中的下一个元素，并记住当前执行的位置和状态。
  • yield break;：立即终止迭代，发出序列结束的信号。

关键应用：Unity协程。

extern

用于声明在外部（非托管代码）实现的方法。

  • 告诉 C# 编译器，某个方法的实现不在当前程序集中，而是位于外部的非托管代码（如 DLL）中。
  • 允许 C# 代码调用 C、C++ 等语言编写的本地库函数，或操作系统提供的 API

常见用法是在使用 Interop 服务调入非托管代码时与 DllImport 特性一起使用。 在这种情况下，还必须将方法声明为 static

readonly

用于字段

对字段的赋值只能作为声明的一部分或在同一类的构造函数中发生。

  • 值类型是不可变的。
  • 引用类型必须始终引用同一对象。但是不保证对象本身不可变。
readonly struct
readonly实例成员

可以使用 readonly 修饰符来声明实例成员不会修改结构的状态。
这个只限于struct可用。struct是值类型，而class是引用类型。class始终可以修改对象状态，没有readonly限制。

和C++相比较的题外话

特性Attribute

特性Attribute用于添加元数据。如编译器指令和注释、描述、方法、类等其他信息。
这些元数据可以在运行时通过反射（Reflection）读取，用于影响程序行为、实现动态逻辑或与框架功能集成。

特性（Attribute）的名称和值是在方括号内规定的，放置在它所应用的元素之前。positional_parameters 规定必需的信息，name_parameter 规定可选的信息。

  • 所有特性类必须直接或间接继承自 System.Attribute。
  • 通过反射读取特性信息，动态调整程序行为。

比如Unity中菜单按钮的创建

反射Reflection

访问、检测和修改它本身状态或行为的一种能力。
程序集包含模块，而模块包含类型，类型又包含成员。反射则提供了封装程序集、模块和类型的对象。

允许在运行时（Runtime） 动态地：

  • 获取程序集、模块、类型的元数据（信息）。
  • 创建类型的实例。
  • 调用类型的方法、属性、字段。
  • 访问和操作特性（Attributes）。

反射将代码中的类型、方法等都抽象为对象（如 Type, MethodInfo, PropertyInfo），使得程序可以在运行时像操作普通数据一样来分析和操作代码本身。

反射基于System.Reflection命名空间下的一系列类来实现。

属性Property

类和结构体中封装数据的成员。定义类成员的访问和设置规则，通常用于隐藏字段（Fields）的内部实现细节，同时提供控制数据访问的机制。

可以看作是对字段的包装器，通常由 get 和 set 访问器组成。它们具有可读写或计算它们值的 访问器（accessors）。

抽象属性 - 抽象类可以具有抽象属性，属性需要在派生类中实现。

事件Event

一种成员，用于将特定的事件通知发送给订阅者。事件通常用于实现观察者模式，它允许一个对象将状态的变化通知其他对象，而不需要知道这些对象的细节。

C# 中使用事件机制实现线程间的通信。

事件使用 发布-订阅（publisher-subscriber） 模型。

关键点：

  • 声明委托：定义事件将使用的委托类型。委托是一个函数签名。
  • 声明事件：使用 event 关键字声明一个事件。
  • 触发事件：在适当的时候调用事件，通知所有订阅者。
  • 订阅和取消订阅事件：其他类可以通过 += 和 -= 运算符订阅和取消订阅事件。

事件（Event） 是一个受保护的、封装了的委托实例成员。它对外提供了一个安全的订阅（+=）和取消订阅（-=）机制，但通常不允许外部直接触发（Invoke()）或覆盖（=）。

在这里，订阅者需要注意，销毁时要处理订阅的取消，否则可能导致对象仍然被事件持有引用，不会被GC。

闭包

两种方式都可以捕获（访问） 定义它们所在作用域内的外部变量（局部变量、参数、字段）。这被称为闭包（Closure）。

闭包就是一个 “自带环境的函数” 。它不仅包含要执行的代码块，还“包裹”了它被定义时所处的上下文（即当时可访问的那些局部变量）。

本质

编译器自动创建了一个辅助类来封装被捕获的变量和函数。
函数原型+上值引用+全局环境引用

注意事项
  • 被捕获的变量生命周期被延长了，会一直存活到委托被垃圾回收为止，而不是在方法执行完就销毁。留意GC问题。
  • 多个委托可以捕获同一个变量，操作的是同一个变量实例。（引用问题）
  • 在Unity中要极其小心：如果在协程或异步回调中捕获了循环变量（如 for (int i=0; i<10; i++)），通常会产生bug，因为当回调执行时，循环早已结束，i 的值已经是最终值（通常是10）。解决方法是在循环内创建一个局部副本。（引用问题）

下面列出了线程生命周期中的各种状态：

  • 未启动状态：当线程实例被创建但 Start 方法未被调用时的状况。

线程池 ThreadPool

优点：
性能好：避免了频繁创建销毁线程的开销。
管理简单：由CLR自动管理线程的创建、回收和调度。

缺点：
控制力弱：无法控制池中线程的优先级、名称等属性。
不适合长任务：长时间运行的任务会占用池中的线程，可能影响其他短任务的调度。

与async/await集成：编写异步代码就像写同步代码一样简单直观。

关于C、C++和C#的异同和发展历史

C、C++和C#区别概述

C

C也是一门低级的编程语言，它的效率和汇编接近。它提供了基本级（base-level）的访问内存方式，对运行时支持的要求非常小。

C++

增强C语言并在不牺牲速度和效率的前提下为其增加面向对象（object-oriented）的能力。

C++被认为是一种中级的语言，因为它基于低级的C语言所创建并拥有一些高级的能力。

C#

C#将代码编译成字节码（byte-code）而不是机器码（0101的二进制）。这意味着它会在一台虚拟计算机上运行，这台虚拟机实时（on the fly）将字节码翻译成机器码。它在C语言的基础上添加了垃圾回收（garbage collection）、未初始化的变量检查（uninitialized variable checking）、边界检查（bound checking）和类型检查（type checking）这些特性。

C vs C++

C语言是不支持面向对象的，这也是它和C++的最大的区别。C是纯过程式（procedural）编程语言，而C++是既支持过程式又支持面向对象的混合编程语言。

其他的一点

  • C++使用名称空间（namespace）将代码组织为逻辑组并阻止了名称冲突（collision）。当你的代码基（code base）包含了多个库的时候会发生名称冲突。C不使用名称空间。
  • C使用函数来实现输入和输出，而C++使用对象来实现输入和输出。
  • C++支持引用变量（reference variable），而C不支持。引用变量是已存在变量的别名，可用于加速编码。
  • C不支持错误和异常处理，但是C++支持。

C++ vs. C#

C++是一门面向对象的语言，而C#被认为是一门面向组件（component）的编程语言。面向对象编程聚焦于将多个类结合起来链接为一个可执行的二进制程序，而面向组件编程使用可交换的代码模块（可独立运行）并且你不需要知道它们内部是如何工作的就可以使用它们。

C# 通常被认为是 编译型语言，但它的执行过程融合了编译和即时编译的特点。
代码在运行前必须经历一个明确的、由编译器执行的编译步骤（从 C# 到 IL）。
不过后面由提前编译，AOT(Ahead of time)编译，使用像 Native AOT 发布选项时，C# 代码会被直接编译成特定平台的原生机器码。

关于AOT

以下是C++和C#的主要区别。

  • C++将代码编译成机器码，而C#将代码编译成中间语言IL，由CLR（Common Language Runtime）运行时JIT编译为机器码。(Just In Time即时编译器JIT编译器会在代码执行期间，分析并编译那些频繁执行的“热代码”部分，将其转换为更高效的机器码，从而提升程序性能。)
  • C++要求用户手动处理内存，但是C#运行在虚拟机中，而虚拟机会自动处理内存。(自动内存管理)
  • C#不使用指针（pointer），而C++可以在任何时候使用指针。
    虽然C++的设计是用于Unix或类Unix的系统，但是它可以运行在任何的平台上。C#虽然已标准化，但是基本上只在Windows的环境下运行。
  • C++可以创建独立（stand-alone）和控制台（console）程序。C#可以创建控制台，Windows，ASP.NET和移动（mobile）程序，但不能创建独立程序。
