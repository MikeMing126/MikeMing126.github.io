Resource Management

发表于2025-09-05|更新于2025-09-05|Unity
|总字数:1.6k|阅读时长:5分钟|浏览量:

资源Asset

An asset is any file you use in your project

  • Visual elements: 3Dmodels, textures, sprites.
  • Audio elements
  • Abstract items: 颜色渐变、动画蒙版、文本、数字数据等

Unity在进行资源导入时，会对大多数类型的资源进行处理，从其原始格式转为适合创作时和运行时的格式。

关于Sprite和Texture:

Texture是基础纹理，适用于大图。Sprite则专门为2D和UI设计，通常用于Image。

If you’re used to working in 3D, sprites are similar to standard textures but there are special techniques to combine and manage sprite textures for efficiency during development.

RawImage组件支持Sprite和Texture。而Image组件则只支持Sprite。

图集的概念，准确说时Sprite Atlas。也是针对Sprite的。其将多个纹理合并为一个组合纹理的资源。Unity 可以调用此单个纹理来发出单个绘制调用而不是发出多个绘制调用，能够以较小的性能开销一次性访问压缩的纹理。

Asset Bundle & Asset Packages

资产分组：
Asset Bundle: Group assets together in an archive file format.
用于远程更新资产、提供DLC等。

资产打包：
Asset Packages
用于在不同Unity项目之间共享。

Unity原生资源管理

  • Asset Database API 来执行加载、导入、卸载等操作。
  • Resources System 另一种管理资源加载的方式。会影响程序性能
  • Addressables package 提供了用于管理运行时资产加载的简化工作流程 Unity推荐使用的系统。需要安装对应的Package才能使用。

Asset Database

Unity在进行资源导入时，会对大多数类型的资源进行处理，从其原始格式转为适合创作时和运行时的格式。
原始格式称之为源资源(source asset)，对应的导入文件为Artifact。

Artifacts have a serializable C# class representation that inherits from UnityEngine.Object, so you can access their properties and work with them in code.

因此对应导入的Artifact实际上也是UnityEngine.Object。

每个Artifact的文件名称都是一个没有文件拓展名的GUID。
相关的资产数据库信息位于Library文件夹中。Artifact是缓存数据，Unity可以基于项目的保存导入设置从源文件重新生成。

在依赖数据、源文件改变时，会重写导入并更新。相关的操作较为耗时，Unity有Unity Accelerator解决方案来优化处理。

相关编程操作

以Unity编辑器相同的方式访问和操作资源。创建、导入、删除、复制、移动、加载和保存资源，以及搜索资源数据库。

一般用于编辑器窗口和编辑器脚本

运行时资产操作

使用AssetBundle动态加载资产，使用Addressables package减少构建大小和发布内容更新。

Unity Build时包含的资产：

  • Scene List中的Scenes
  • Preferences设置
  • 直接引用资产
  • （Resource System）所有位于Resources文件夹的资源
  • （AssetBundle）：Any assets and scenes assigned to an AssetBundle, plus any direct references from those assets or scenes.
  • (Addressables):any assets and scenes assigned to an Addressable group, plus any direct references from those assets or scenes.

运行时资源管理系统：

  • Direct Reference
  • Resource System
  • Asset Bundle
  • Addressable Package(基于Asset BundleAPI构建)
  • ECS content management system: For Entities package.

Resources System

简单的资源管理方式。创建Resources文件夹。

During the build process, Unity finds the assets in the Resources folder and bundles them into a serialized file with metadata and indexing information.

  • 不管是否引用都会构建
  • 不支持增量，需要全量构建。
  • Resources.Load会一直驻留内存中，除非调用Resources.UnloadUnusedAssets明确卸载。或者直到使用他们的场景卸载。

Asset Bundle

单独的资源容器 - Asset Bundle。创建downloadable content(DLC)
Asset Bundle API支持按需下载。

Asset Bundle是平台特定的。不同平台的AssetBundle不同。在传输过程中，还可以进行LZ4、LZMA进行压缩。

内容：
AssetBundle - Container file formmat.
二进制头文件包含以下文件类型：

  • 序列化文件[Serialized File]
    • Only Asset: A Single Serialized File.
    • Only Scenes: 会创建两个序列化文件，一个包含场景层级结构中的对象，另一个包含所有引用的对象
  • 资源文件[Resource File]
    包含为特定资源单独存储的二进制数据块。

The AssetBundle file always includes a serialized AssetBundle object which acts like a directory for the contents of the AssetBundle.

限制：

  • 不支持前向兼容。通常和新版本兼容。
  • 不允许包含程序集。但可以用于预编译的对象实例，如ScriptableObject Assets。
  • 仅能通过Script使用
  • Asset Bundle API不会跟踪资源的引用关系。需要自己处理AssetBundle之间的依赖关系。

    Safely unloading AssetBundles often requires writing a reference counting system

  • 需要手动分配和释放内存
  • API本身不知道对应AssetBundle位于本地还是远程，需要自己跟踪

Addressables package

在Unity编辑器中提供了一个用户界面，用于管理和组织项目中的资源。它基于 AssetBundle API 构建，可自动管理依赖项、资源位置和内存分配。旨在消除资源和 AssetBundle 系统的限制，让您能够更轻松地按需管理资源。

可寻址管理。使用地址来处理资源，而不是文件名/位置。

Once you have made an asset Addressable, you can reference it by address in your code, rather than by its file name or AssetBundle location, which means you can change the location of an asset without needing to rewrite any code.

ECS content management system

Entities package 拥有自己的内容管理系统 ，您可以在其中使用对资源的弱引用在运行时加载它们。您还可以创建内容存档，以便将内容传递给应用程序。

YooAsset

AssetBundle本身需要手动分配和释放内存，也需要自己维护引用相关信息。YooAsset便是一个资源管理系统。协助处理相关的逻辑。

特点:

  • 同步异步加载、远程下载、多线程下载、下载器等
  • 可寻址资源定位
  • 分包打包、引用计数等
